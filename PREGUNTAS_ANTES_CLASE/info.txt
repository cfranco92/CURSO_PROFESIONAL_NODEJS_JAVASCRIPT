-->¿Por qué es importante aprender Node.js y Javascript?
   -----------------------------------------------------
   * JavaScript es un lenguaje que gobierna actualmente las aplicaciones web,.
   * Sitios de eccommerce están basados hoy en día en JavaScript en el lado del
     servidor.
   * Se han dado otros desarrollos como internet de las cosas.
   * Tiene uso desde hace muchos años. Hoy en día se puede usar en servidores,
     internet de las cosas.
   * Todos los proyectos hoy por hoy tienen una parte en la web y JavaScript es
     fundamental para esto.

-->Explícanos el cambio de io.js a Node.js
   * Node es un proyecto de código abierto.
   * Vino un grupo de gente con las políticas de Node.js y cambiaron su desarrollo
     por otro lado como io.js. Los últimos años hemos tenido una unión (merge)
     entre io.js y Node.js quedando con el nombre de nuevo Node.js bajo Node
     Fundation.

-->Qué tiene que saber alguien que va a tomar este curso?
   * Se asume que tiene una idea de desarrollo web, protocolo http y programación
     en general.
   * El que no tenga experiencia previa, se le puede escapar algunas cosas.


-->Qué herramientas vamos a utilizar?
   * Se va utilizar sublime text o atom. Igualmente Node no impone ningún editor
     de texto. Que mínimamente tenga resaltador sintáctico.
   * A lo largo del curso se dirá dónde descargar NodeJS.
   * Se recomienda utilizar Linux y NodeJS. En Windows en linea de comandos puede
     ser un poco diferente.

-->Temario y proyecto
   * Se parte de una aplicación realizada previamente una gran parte.
   * Se va a poner una aplicación servidor, donde podamos tener series que se
     guarden en la base de datos, utilizando MongoDB.
   * La idea es no depender de una API externa.
   * Funciones sincrónicas y asincrónicas.
   * Crear un server muy básico.
   * Express.
   * MongoDB.

-->Por qué elegir Node.js, que me ofrece?
   * Node se inventa para manejar de forma rápida request http, poder manejar
     muchas request. Esto se resuelve un libu, cada request se va encolando.
     Es muy util para operaciones muy costosas.
   * El acceso de a la base de datos que puede demorar mucho tiempo, esto no va
     ha bloquear el acceso desde otros puntos.
   * Resuelve el tema de concurrencia.

-->En qué casos no deberíamos usar Node.js?
   * Operaciones que requieran mucho cálculo. Se puede utilizar, pero no es para
     lo que se inventó.
   * Cabe recordar que NodeJS puede utilizar hilos.

-->Con tecnologías como python/django, ruby/ROR, php/laravel, javascript/nodejs, etc.. ¿cual recomiendas para desarrollarse profesionalmente?
   * Todo depende, pero actualmente se puede aprovechar mucho el desarrollo web
     utilizando javaScript y NodeJS en el lado servidor.

-->Vamos a utilizar EMACScript 6?

     babel.

-->Trabajaremos con el MEAN Full-Stack?
   * Mongo, Express, Angular y Node. Se verán todos menos Angular.

-->Hay servers gratuitos para probar Node?
